E-2000 EnerCharge Pro蓄電システムのBMSアーキテクチャ図は、3階層のバッテリー管理システム構成、物理的なコンテナ内配置、電力変換システム、および運用上の温度制約と性能特性を詳細に示しています。

## E-2000 EnerCharge Pro BMS アーキテクチャ記述

### 1. 構成要素の識別と配置関係

本図は、BPS Corporation製のE-2000 EnerCharge Pro蓄電システムのBMSアーキテクチャを示しており、主要な構成要素は20ftコンテナ（IP55準拠）内に収容されています。

**物理的構成要素:**
*   **20ft Container (IP55)**: システム全体を格納する筐体で、IP55の保護等級を持つ。
*   **バッテリーモジュール**: LFP 14S8P 51.2V仕様のモジュールが計16個（Module 01〜16）配置されている。これらは4列4段で配置されており、下段のModule 13〜16は「higher thermal load (bottom row)」として高温リスクモジュール（35〜45℃）として識別されている。各モジュールには温度センサーが2つずつ（合計32個）配置されている。
*   **冷却ファン**: FAN Fan #1〜#4の計4基がバッテリーモジュール群の下部に配置されており、強制空冷システムを構成する。
*   **PCS (Power Conversion System)**: 250kWの双方向型電力変換システム。SiC MOSFETを採用し、DC700-900VからAC400Vへの変換を行う。Grid / Loadへの接続を示す。

**BMSアーキテクチャ構成要素:**
*   **System BMS (SBMS)**: 最上位のシステムBMS。全体の制御、PCSとの連携、SCADA（Modbus TCP）との通信、SOC/SOH（State of Charge/State of Health）の集約、アラーム管理を担当する。
*   **BMU Group (Battery Management Unit Group)**: 4つのグループに分かれており、それぞれが4つのバッテリーモジュールを管理する。
    *   BMU Group 1: Module 01〜04を管理。SOC/SOH計算とセルバランシングを行う。
    *   BMU Group 2: Module 05〜08を管理。SOC/SOH計算とセルバランシングを行う。
    *   BMU Group 3: Module 09〜12を管理。SOC/SOH計算とセルバランシングを行う。
    *   BMU Group 4: Module 13〜16を管理。SOC/SOH計算とセルバランシングを行う。

### 2. 信号・データの流れ

本図における信号・データの流れは、主にBMSの階層構造と電力の流れによって示されます。

*   **BMS内部のデータフロー**:
    *   各バッテリーモジュール（Module 01〜16）は、それぞれを管理するBMU Group（BMU Group 1〜4）へ、温度センサーを含む監視データを提供する。
    *   各BMU Groupは、担当するモジュール群から得られたデータに基づき、SOC/SOHの計算とセルバランシングの制御を行う。
    *   各BMU Groupは、処理したデータと状態情報をSystem BMS (SBMS)へ送信する。
    *   System BMS (SBMS)は、各BMU Groupからの情報を集約し、システム全体のSOC/SOHを算出し、アラーム管理を行う。
*   **システム外部との連携**:
    *   System BMS (SBMS)は、PCSと連携し、電力変換システムの動作を制御する。
    *   System BMS (SBMS)は、SCADAシステム（Modbus TCPプロトコルを使用）と接続され、上位監視システムへのデータ送信および上位からの制御指示の受信を行う。
*   **電力フロー**:
    *   PCSはバッテリーモジュール群と電気的に接続され、DC700-900Vのバッテリー電圧をAC400Vに変換し、Grid / Loadとの間で双方向の電力供給・受電を行う。

### 4. 技術的含意

この図面から読み取れる設計思想、既知制約、および拡張可能性は以下の通りです。

**設計思想:**
*   **スケーラブルな階層型BMS**: 多数のバッテリーモジュールを効率的かつ安全に管理するため、モジュールレベルのBMUとシステム全体のSBMSを組み合わせた階層構造を採用。これにより、システム全体の複雑性を管理しつつ、各モジュールの詳細な監視と制御を可能にしている。
*   **安全性と信頼性への配慮**: IP55準拠のコンテナ、LFPセルの採用、多点温度監視（32個のセンサー）、および高温時の出力制限は、システムの安全性と長期的な信頼性を確保するための重要な設計要素である。特に、高温時の性能低下を明示することで、運用上のリスクを明確にしている。
*   **熱管理の最適化**: 下段モジュール（13-16）の熱負荷が高いことを明記し、4基のファンによる強制空冷システムを導入していることから、熱管理がシステムの性能と寿命に与える影響を重視していることが示唆される。

**既知制約:**
*   **高温環境下での性能制限**: 35〜45℃の高温環境では、充放電の最適化が行われず、出力が75%に制限される。また、SOC精度が低下し、サイクル寿命保証が25℃運用時のみとなる。これは、高温環境での運用がシステムの性能と寿命に直接的な影響を与えることを示している。
*   **空冷能力の限界**: 周囲温度40℃以上での空冷に限界があることが示唆されており、設置環境の温度条件が重要となる。
*   **特定のモジュールの熱負荷**: Module 13〜16が他のモジュールよりも熱負荷が高いという設計上の特性があり、これらのモジュールの監視と管理が特に重要となる。

**拡張可能性:**
*   **SCADA連携による統合**: Modbus TCPを介したSCADAシステムとの連携は、より大規模なエネルギー管理システムやグリッド運用システムへの統合を容易にする。
*   **ファームウェアによる機能改善**: 「FW v3.0 candidate」の記載は、将来的なファームウェアアップデートによって、システムの性能向上、新機能追加、または既存制約の緩和が期待できることを示唆している。
